## Why Meerschaum?

Meerschaum solves the most painful part of time-series ETL: incremental updates. Instead of writing custom logic to fetch only new records, Meerschaum tracks the last sync point for each pipe and automatically fetches only the data you need.

<CardGroup cols={2}>
  <Card title="Incremental sync" icon="rotate">
    Meerschaum calculates the correct time window for each sync, fetching only new or updated data — no full reloads.
  </Card>
  <Card title="Any data source" icon="plug">
    Write a simple `fetch()` function that returns a DataFrame, and Meerschaum handles storage, deduplication, and indexing.
  </Card>
  <Card title="Multiple backends" icon="database">
    Store data in PostgreSQL, TimescaleDB, MySQL, SQLite, MSSQL, DuckDB, or your own SQL database.
  </Card>
  <Card title="Production ready" icon="shield-check">
    Background jobs, systemd integration, web API, and a Grafana stack — everything you need for production.
  </Card>
</CardGroup>

## Quick Install

```bash
pip install -U meerschaum
mrsm stack up -d db grafana
mrsm bootstrap pipes
```
