## What is Meerschaum?

Meerschaum is a tool for quickly synchronizing time-series data streams called **pipes**. With Meerschaum, you can have a data visualization stack running in minutes.

The core idea is simple: define where your data comes from, and Meerschaum handles **incremental updates** automatically. It fetches only the data you need, deduplicates rows, and upserts changes — so your pipelines stay fresh without full reloads.

## Why Meerschaum?

If you've worked with time-series data, you know the headaches that come with ETL. Data engineering often gets in analysts' way, and every minute spent on pipelining is time taken from real analysis.

Rather than copy-pasting ETL scripts, build pipes with Meerschaum and let the framework handle the rest.

## Key Concepts

### Pipes

A **pipe** is a data stream defined by three labels:

- **Connector** — where the data comes from (e.g., `plugin:noaa`, `sql:main`)
- **Metric** — what the data represents (e.g., `weather`, `sales`)
- **Location** — an optional tag for grouping (e.g., `nyc`, `atl`)

```python
import meerschaum as mrsm

pipe = mrsm.Pipe('plugin:noaa', 'weather', 'atl', instance='sql:local')
```

### Connectors

**Connectors** define database connections and data sources. Meerschaum supports SQL connectors (PostgreSQL, SQLite, etc.), API connectors, and custom plugin connectors.

### Plugins

**Plugins** extend Meerschaum with new data sources. A plugin is a Python file with a `fetch()` function that returns a DataFrame.

### Instances

An **instance** is where pipe metadata and data are stored. The default is `sql:main` (a local SQLite database).
