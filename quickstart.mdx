## Prerequisites

- Python 3.8 or higher
- `pip` package manager
- Docker (optional, for the TimescaleDB + Grafana stack)

## Install Meerschaum

<Steps>
  <Step title="Install the package">
    Install Meerschaum from PyPI:

    ```bash
    pip install -U --user meerschaum
    ```

    Verify the installation by running:

    ```bash
    mrsm --version
    ```
  </Step>

  <Step title="Start the Docker stack (optional)">
    If you want a local TimescaleDB database and Grafana for visualization, start the built-in stack:

    ```bash
    mrsm stack up -d db grafana
    ```

    This starts:
    - **TimescaleDB** on port 5432 (default `sql:main` instance)
    - **Grafana** on port 3000 (default login: `admin` / `admin`)

    <Note>
    If you skip this step, Meerschaum will use a local SQLite database (`sql:local`) instead.
    </Note>
  </Step>

  <Step title="Install a data source plugin">
    Install the NOAA weather plugin to fetch live weather data:

    ```bash
    mrsm install plugin noaa
    ```
  </Step>

  <Step title="Bootstrap your first pipe">
    Use the interactive wizard to create a pipe:

    ```bash
    mrsm bootstrap pipes
    ```

    When prompted:
    - **Connector**: `plugin:noaa`
    - **Metric**: `weather`
    - **Location**: `atl` (Atlanta Hartsfield-Jackson airport code: `KATL`)

    Or skip the wizard and register directly:

    ```bash
    mrsm register pipe -c plugin:noaa -m weather -l atl -i sql:local
    ```
  </Step>

  <Step title="Sync your pipe">
    Fetch data and create the database table:

    ```bash
    mrsm sync pipes -l atl -i sql:local
    ```

    Meerschaum will fetch weather data from NOAA and store it in `sql:local`.
  </Step>

  <Step title="View your data">
    Show a preview of the synced data:

    ```bash
    mrsm show data -c plugin:noaa -m weather -l atl -i sql:local
    ```

    Or run a SQL query:

    ```bash
    mrsm sql sql:local -q "SELECT * FROM plugin_noaa_weather_atl LIMIT 10"
    ```
  </Step>
</Steps>

## Python API Quickstart

You can also manage pipes programmatically:

```python
import meerschaum as mrsm

# Create a pipe backed by the local SQLite database
pipe = mrsm.Pipe(
    'foo', 'bar',               # Connector and metric labels
    target   = 'MyTableName',   # Table name (defaults to 'foo_bar')
    instance = 'sql:local',     # Built-in SQLite DB
    columns  = {
        'datetime': 'dt',       # Datetime index column
        'id'      : 'id',       # ID index column (optional)
    },
)

# Sync data — creates the table and indices on first run
pipe.sync([{'dt': '2022-07-01', 'id': 1, 'val': 10}])

# Duplicate rows are ignored
pipe.sync([{'dt': '2022-07-01', 'id': 1, 'val': 10}])
assert len(pipe.get_data()) == 1

# Rows with matching keys are updated (upserted)
pipe.sync([{'dt': '2022-07-01', 'id': 1, 'val': 100}])
assert len(pipe.get_data()) == 1

# Query data with time range and value filters
df = pipe.get_data(
    begin  = '2022-01-01',
    end    = '2023-01-01',
    params = {'id': [1]},
)
# Returns:
#           dt  id  val
# 0 2022-07-01   1  100

# Remove the pipe and its table
pipe.delete()
```
